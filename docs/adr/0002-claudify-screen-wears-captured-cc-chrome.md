# The Claudify screen replicates Claude Code's dialog chrome, captured live

The Claudify screen has no Claude Code counterpart, and the project's original scope said pi-only surfaces keep pi's native styling. We deliberately amended that: the screen replicates the exact chrome of Claude Code's `/config`, `/mcp`, and `/theme` dialogs (borders, spacing, key-hint footer, colors, esc semantics), captured live per the capture-don't-guess method, with our settings as the content. The pi-native-skin alternative was rejected because the package's whole identity is that pi feels like Claude Code, and a settings screen in a different skin breaks that feeling. Consequence: live captures of those three dialogs (including narrow-width behavior) are a prerequisite for building the screen, and pi's stock `SettingsList` theming will be overridden.
